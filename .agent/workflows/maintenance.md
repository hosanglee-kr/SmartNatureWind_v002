# /maintenance - 규칙 유지보수 워크플로우

이 워크플로우는 `.cursorrules`에 정의된 구현 및 명명 규칙을 준수하도록 코드를 관리하고, 규칙 자체를 개선하는 절차를 설명합니다.

## 1. 규칙 준수 여부 점검 (Audit)
수정되거나 새로 생성된 파일에 대해 다음 명령을 실행하여 규칙 준수 여부를 확인합니다.

1. 열려 있는 파일의 코드를 분석합니다.
2. `.cursorrules`의 네이밍 규칙(접두사, 변수명 길이)과 구현 규칙(로깅, 메모리 처리 등)을 대조합니다.
3. 위반 사항이 있다면 `[위반]` 태그와 함께 리스트업합니다.

## 2. 코드 보완 (Refactor)
점검 결과 위반 사항이 발견되면 다음과 같이 수정합니다.

1. **변수명**: 의미가 불분명한 짧은 변수(`v_doc`, `v_r` 등)를 긴 이름(`v_jsonDoc`, `v_randomValue`)으로 변경합니다.
2. **로깅**: `Serial.print` 등을 전용 로거(`CL_D10_Logger`)로 교체합니다.
3. **주석**: 복잡한 로직에 `// 1. ...`, `// 2. ...` 단계별 주석을 추가합니다.

## 3. 규칙 고도화 (Improvement)
AI가 특정 오류를 반복하거나 새로운 코딩 패턴이 필요할 경우 규칙을 업데이트합니다.

1. `.cursorrules` 파일을 엽니다.
2. 새로운 규칙을 명확한 예시와 함께 추가합니다.
3. 규칙의 버전을 상단에 명시합니다 (예: `v2.x`).
